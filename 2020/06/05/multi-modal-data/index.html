Avito-kaggle引入

Avito是kaggle上一个关于商品供求的多模态数据集，要求以文本数据（商品的标题、描述等文本信息）图像数据（商品图片）以及结构化数据（历史需求数据等）为输入源，预测商品需求，以使得卖家可以有方向性地更好地优化自己的商品列表，以免投入过多的成本。

First place solution

dance with ensemble

KeyPoints: feature engineering, NN, xgb, lgb, stacking(build diversified models)

FrameWork:

  • First layer: lgb, NN, xgb;

  • Second layer: lgb, xgb, NN

  • Final layer: NN

NN

Exploring procedure:

Considering text, categorical, numerical, images features.

  • Baseline: considering numerical features and categorical embedding
  • including title and description with RNN, with fastText pretrained input embedding. Self training fastText embedding on train+test
  • including image features, with VGG16 top layer. set a separate layer before merging text, image, categorical, numerical features togethre
  • try different models for text and image: Text models: LSTM, CNN, Attention; Image models: ResNet50 middle layer(what is middle layer?)
  • Fine-tuning: dropout, BNs
  • considering more features
  • trick: save models along the way and train a fully connected NN on top of them(kind of bagging somehow)

Summary:

  • For categorical variables: embedding each categorical, and then using dense layer to aggregate these dense vectors.
  • For numerical fearues: feature engineering is important.
  • For image data: using pretrained models to extract features.
  • For Text data: using FastText to embed text data, and then using LSTM to extract features.

XGB and LGB

Feature engineering:

  • text features: tf-idf for all xgb models, svd and oof ridge features(what’s that??) for all lgb models
  • image features: image statistics(measuring image quality: dulless, whiteness, average pixel width, dominant/average color of the image , width/height/size, blurrness); features from ResNet 50, InceptionV3 and Xception(including the predicted categories as new feature), features from vgg16, 512 raw features+first 15 PCA features, keypoint features(imagetop1, some kind of classification result, **how to build a similar feature with significance? **).
  • categorical features: count unique features; target encoding at various levels(mean deal probability levels)
  • predicted independent variables features: predicted independent variables features with xgb or lgb, and then give the mean features of these variables above according to categorical levels; also include the diff features between the predicted ones and the true variables at various levels.
  • user_id features: (may try user2vec) treat userid as categorical variable, generate features at (userid) or (userid, other categorical variables) level(user_id interactions with other variable)
  • while generating new features with calculating between other existing features, we often use lag/log or other functions for some simple transformation. (calculating min max mean sd or other statistics to be new features at categorical level may be useful. training some intermediate models to generate features may be useful.)

second place solution

特征角度：

image features：用预训练模型VGG16、ResNet50、MobileNet做图片特征提取

text features：在整个数据集上训练FastText做embedding，包括描述、城市title、类别title等

statistical features：比较重要。一级二级三级品类的平均价格

无监督学习：品类特征上采用自编码模型分离特征。采用user2vec对user_id编码

模型上：

神经网络：FM like style for categorical features with embeddings？？数值特征、fasttext vectors、image vectors、BiLSTM for words、BiLSTM for characters with max, avg poolings with attention、target encoded features for categorical features and second and third order interactions， user2vec features. With cyclic LR, adam optimizer, BNs, dropouts.

LGB: fasttext vec 对lgb单个模型很好。 可以考虑SVD over TFIDF
